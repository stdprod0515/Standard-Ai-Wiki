# Git CLI 사용법: 파일 동기화의 원리

안녕하세요, 보스님! 저는 보스님의 AI 비서 자비스입니다. 오늘은 보스님만의 지식 저장소를 구축하는 데 매우 유용한 'GitHub'에 대해 안내해 드리겠습니다.

## 1. GitHub란 무엇인가요?

*   **어떤 사이트인가요?**
    GitHub는 전 세계 개발자들이 소프트웨어 코드를 공유하고, 협업하며, 버전을 관리하는 데 사용하는 **가장 크고 인기 있는 웹 기반 플랫폼**입니다. 마치 코드를 위한 소셜 네트워크이자, 협업 도구라고 생각하시면 됩니다.

*   **주요 목적:**
    *   **코드 저장 및 관리:** 소스 코드, 문서, 설정 파일 등 프로젝트 관련 모든 파일을 안전하게 저장하고 관리합니다.
    *   **버전 관리 (Git):** 'Git'이라는 시스템을 사용하여 파일의 변경 이력을 체계적으로 기록하고 관리합니다. 과거 버전으로 돌아가거나, 특정 변경 사항을 추적하기 용이합니다.
    *   **협업:** 여러 사람이 한 프로젝트에 동시에 참여하여 각자의 작업을 공유하고, 다른 사람의 작업 내용을 검토하며 협력할 수 있도록 돕습니다.
    *   **프로젝트 공유:** 자신의 프로젝트를 전 세계 사람들과 공유하거나, 다른 사람들의 공개된 프로젝트를 살펴보고 활용할 수 있습니다.

## 2. 왜 GitHub를 사용하나요? (보스님께 특히 유용한 점)

*   **나만의 지식 저장소 구축:** 앞서 대화에서 말씀드렸듯이, 제가 정리해 드린 정보들을 마크다운 파일(`.md`)로 만들어 GitHub 저장소에 올리시면, 이것이 보스님만의 **개인화된 지식 센터**가 됩니다. 언제든 필요할 때 찾아볼 수 있고, 내용을 수정하거나 추가하여 계속 업데이트할 수 있습니다.
*   **안전한 백업:** 중요한 정보나 작업 결과물을 GitHub에 저장해두면, 로컬 PC에 문제가 발생하더라도 데이터를 안전하게 보존할 수 있습니다.
*   **체계적인 기록 관리:** 대화 내용, 프로젝트 진행 상황, 학습한 내용 등을 날짜별, 주제별로 정리하여 GitHub에 저장하면, 나중에 특정 정보를 찾거나 작업 과정을 되짚어보는 데 매우 유용합니다.
*   **쉬운 공유:** 만약 보스님께서 다른 사람과 정보를 공유해야 할 경우, GitHub 저장소를 통해 쉽고 안전하게 공유할 수 있습니다.

## 3. GitHub 사용 방법 (초보자 가이드)

GitHub를 처음 사용하신다면, 몇 가지 기본적인 단계를 따라 하시면 됩니다.

**Step 1: GitHub 계정 생성**

1.  [GitHub 웹사이트](https://github.com/)에 접속합니다.
2.  "Sign up" 버튼을 클릭하여 계정을 생성합니다. 이메일 주소, 사용자 이름, 비밀번호 등을 입력해야 합니다.
3.  이메일 인증 절차를 완료하면 계정 생성이 완료됩니다.

**Step 2: 첫 번째 저장소(Repository) 만들기**

*   **저장소란?** 저장소는 특정 프로젝트의 모든 파일과 그 변경 이력을 담는 공간입니다. 보스님께서 저와의 대화 내용을 저장할 저장소를 하나 만드시면 됩니다.
*   **만드는 방법:**
    1.  GitHub에 로그인한 후, 우측 상단의 프로필 아이콘 옆에 있는 "+" 버튼을 클릭합니다.
    2.  "New repository"를 선택합니다.
    3.  **Repository name:** 저장소 이름을 입력합니다. (예: `my-ai-assistant-notes`, `knowledge-base` 등)
    4.  **Description:** 저장소에 대한 간단한 설명을 추가할 수 있습니다. (예: "자비스와 나눈 대화 및 AI 학습 내용")
    5.  **Public / Private:**
        *   `Public`: 누구나 저장소의 내용을 볼 수 있습니다. (초기에는 Private으로 설정하는 것을 권장합니다.)
        *   `Private`: 보스님만 저장소의 내용을 볼 수 있습니다.
    6.  **Initialize this repository with a README:** 이 옵션을 체크하면, 저장소를 만들면서 자동으로 README.md 파일이 생성됩니다. README 파일은 저장소에 대한 설명이나 사용법 등을 적는 곳으로, 처음 시작할 때 유용합니다.
    7.  "Create repository" 버튼을 클릭하여 저장소를 생성합니다.

**Step 3: 파일 업로드 (예: 제가 드린 마크다운 파일)**

저와의 대화에서 제가 드린 정보들을 마크다운 파일로 저장하셨다면, 이 파일들을 GitHub 저장소에 업로드할 수 있습니다.

1.  **GitHub 저장소 페이지 접속:** 생성하신 저장소 페이지로 이동합니다.
2.  **파일 업로드:**
    *   저장소 페이지에서 "Add file" 버튼을 클릭합니다.
    *   "Upload files" 옵션을 선택합니다.
    *   보스님 PC에 저장된 마크다운 파일들을 드래그 앤 드롭하거나, "choose your files"를 클릭하여 파일을 선택합니다.
3.  **커밋 (Commit) 메시지 작성:**
    *   파일을 업로드한 후, 변경 사항을 저장하기 위한 커밋 메시지를 작성합니다. (예: "Add initial AI notes", "Update knowledge base with Llama 3 info")
    *   "Commit changes" 버튼을 클릭하여 저장합니다.

**Step 4: Git CLI 사용 (조금 더 고급 방법 - 선택 사항)**

많은 사용자들이 GitHub와 Git을 함께 사용합니다. Git CLI를 사용하면 파일을 더 효율적으로 관리하고, 버전 관리를 더욱 강력하게 할 수 있습니다.

1.  **Git 설치:** 보스님 PC에 Git이 설치되어 있지 않다면, [Git 공식 웹사이트](https://git-scm.com/downloads)에서 다운로드하여 설치합니다.
2.  **저장소 복제 (Clone):**
    *   GitHub 저장소 페이지에서 초록색 "Code" 버튼을 클릭하고 URL을 복사합니다.
    *   터미널을 열고, 파일을 저장하고 싶은 폴더로 이동한 후 다음 명령어를 실행합니다.
        ```bash
        git clone [복사한 GitHub 저장소 URL]
        ```
    *   이 명령어는 GitHub 저장소의 모든 파일과 이력을 보스님 PC로 복제합니다.
3.  **파일 수정 및 동기화:**
    *   복제된 폴더 안으로 이동하여 파일을 수정하거나 새 파일을 만듭니다.
    *   변경 사항을 **'스테이징(Staging)'** 단계로 옮겨 Git이 변경 사항을 인식하게 합니다. 이 과정은 Git에게 '이 파일들을 다음 커밋에 포함시킬 거야'라고 알려주는 것입니다.
        ```bash
        git add .
        ```
        이 명령어는 현재 폴더 안의 모든 변경 사항을 스테이징합니다. (특정 파일만 하려면 `git add [파일명]`을 사용합니다.)
    *   그 다음, 스테이징된 변경 사항들을 하나의 묶음(버전)으로 만들어 기록합니다. 이것을 **'커밋(Commit)'**이라고 합니다. 커밋은 특정 시점의 파일 상태를 저장하는 스냅샷과 같습니다. 커밋할 때는 왜 변경했는지 설명하는 메시지를 함께 남겨야 합니다. 이는 나중에 변경 이력을 볼 때 큰 도움이 됩니다.
        ```bash
        git commit -m "Add initial AI notes"
        ```
        `"Add initial AI notes"` 부분에 변경 내용을 요약해서 적습니다. 이는 Git에게 '지금은 AI 노트를 처음 추가하는 변경이야'라고 알려주는 것입니다.

**Step 4: GitHub으로 '푸시(Push)' (최종 업로드)**

*   지금까지 컴퓨터에서 작업하고 Git으로 기록한(커밋한) 변경 사항들을 GitHub 저장소로 보내는 과정입니다. 이것을 **'푸시(Push)'**라고 합니다.
*   이 명령어를 실행하면, 보스님 컴퓨터에서 작업한 내용이 GitHub에 있는 원본 저장소에 반영됩니다.
    ```bash
    git push origin main
    ```
    (`origin`은 GitHub 저장소를 가리키는 별칭이고, `main`은 보통 기본 브랜치 이름입니다. 간혹 `master`일 수도 있습니다.)

**정리하자면:**

1.  GitHub 저장소를 **복제(Clone)**해서 내 PC로 가져옵니다.
2.  내 PC의 복제된 폴더에서 파일을 **수정하거나 추가**합니다.
3.  변경 사항을 Git으로 **추가(Add)**하고 **커밋(Commit)**하여 기록합니다. (이것은 변경 사항을 '임시 저장'하고 '기록'하는 단계입니다. `add`는 어떤 것을 포함할지 지정하고, `commit`은 그 묶음을 저장하는 것입니다.)
4.  기록된 변경 사항을 GitHub으로 **푸시(Push)**하여 동기화합니다.

이 과정을 통해 보스님 컴퓨터에서 작업한 내용이 GitHub에 안전하게 저장되고, 언제든 필요할 때 GitHub 저장소에 접속하여 최신 내용을 확인할 수 있습니다.

먼저 GitHub 계정을 만들고, 첫 번째 저장소를 생성하는 것부터 시작해보는 것은 어떨까요? 제가 필요한 파일들을 마크다운 형식으로 준비해 드릴 수도 있습니다.
